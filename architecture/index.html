    Architecture de Vectra

    Vectra est une solution de RAG (Retrieval-Augmented Generation) d'entreprise conçue pour transformer des données non structurées en connaissances exploitables via une interface de chat intelligente.

    Vue d'Ensemble

    L'architecture suit un modèle distribué composé d'une API réactive, d'un worker asynchrone pour l'ingestion de données, et d'une stack AI multi-fournisseurs.

    graph TD
    User([Utilisateur]) <--> Frontend[Frontend Vue.js/Quasar]
    Frontend <--> API[FastAPI Backend]

    API <--> Postgres[(PostgreSQL)]
    API <--> Redis[(Redis Cache)]
    API <--> Qdrant[(Qdrant Vector DB)]

    API -- WebSocket triggers --> Worker[Background Worker]
    Worker -- Ingestion --> Postgres
    Worker -- Vectorization --> Qdrant

    API -- LLM/Embed --> AI[Gemini / OpenAI / Mistral]
    Worker -- Embeddings --> AI

    API -- Tracing --> Phoenix[Arize Phoenix]

    Composants Principaux

    1. Backend API (FastAPI)

    Le cœur du système, responsable de l'orchestration en temps réel :

    • Gestion des Sessions : Authentification et historique des conversations.
    • Orchestration RAG : Intégration avec LlamaIndex pour le découpage, l'indexation et la recherche.
    • WebSocket Manager : Diffusion en temps réel des réponses et des états de synchronisation.
    • Sémantique Cache : Utilisation de Redis pour stocker les résultats des requêtes similaires et réduire la latence/coûts.

    2. Background Worker

    Un service autonome dédié aux tâches lourdes :

    • Ingestion Multi-Source : Scan et extraction de données depuis diverses sources (Connecteurs).
    • Pipeline de Vectorisation : Transformation documentaire en vecteurs via les modèles d'embedding (Gemini 004).
    • Synchronisation Temps Réel : Connecté à l'API via WebSocket pour réagir instantanément aux demandes de l'utilisateur.

    3. Couche de Persistance

    • PostgreSQL : Stocke les métadonnées, les configurations des connecteurs et la structure documentaire.
    • Qdrant : Base de données vectorielle haute performance pour la recherche sémantique ultra-rapide.
    • Redis : Cache sémantique et stockage temporaire.

    4. Intelligence Artificielle

    Vectra est "Model Agnostic" mais optimisé pour la suite Google Cloud :

    • Modèles de Chat : Gemini 1.5 Pro/Flash pour le raisonnement complexe.
    • Embeddings : Gemini Text Embedding 004 pour une représentation sémantique state-of-the-art.
    • Reranking : Utilisation de modèles de reranking pour affiner la pertinence des résultats.

    Flux de Données (RAG)

    1. Requête : L'utilisateur pose une question via le frontend.
    2. Cache : L'API vérifie dans Redis si une question similaire a déjà été traitée.
    3. Retrieval : Si non, Vectra interroge Qdrant pour extraire les passages les plus pertinents.
    4. Augmentation : Le contexte extrait est injecté dans le prompt de l'LLM.
    5. Génération : L'LLM génère une réponse sourcée et précise.
    6. Streaming : La réponse est renvoyée par morceaux (chunks) via WebSocket pour une expérience utilisateur fluide.

    Observabilité

    Le système intègre Arize Phoenix via le protocole OpenInference, permettant de tracer chaque étape d'une requête RAG, d'analyser la latence et de monitorer la qualité des réponses.
